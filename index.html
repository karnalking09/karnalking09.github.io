Quick Start (Beginner)
  • Understand basic web: HTTP, HTML, JS, cookies, headers
  • Learn Linux basics & command line (bash, ssh, pipes)
  • Start with OWASP Top 10 (in-depth study)
  • Practice on safe platforms (TryHackMe, Hacker101, PortSwigger Academy)
Core Books & Reading
  • OWASP Top 10 — must-read for web bugs
  • "The Web Application Hacker's Handbook" — deep web app attacks
  • "Hacking: The Art of Exploitation" — fundamentals & C
  • "Real-World Bug Hunting" — practical bug hunting writeups
Interactive Learning & Labs
  • TryHackMe — guided rooms and learning paths
  • Hack The Box — labs & pro labs

Bug Bounty Platforms
  • HackerOne — public programs & reports

Essential Tools
  • Burp Suite (Community & Pro) — proxy & active scanning
  • Nmap — network & port scanning
  • ffuf / dirb — directory & fuzzing
  • Sqlmap — automated SQLi testing (use responsibly)
  • Metasploit — exploitation framework (lab use)

Writeups & Reports

Courses & Videos
  • PortSwigger Academy (free)

CTFs & Practice Targets
  • picoCTF, Root-Me, CTFtime for events
  • Practice vulnerable apps: DVWA, Juice Shop
  • Hands-on VMs: TryHackMe rooms, HTB boxes
Communities & Blogs
  • r/netsec, r/bugbounty on Reddit — discussions & hints
  • Blogging sites: PortSwigger blog, HackerOne blog
  • Twitter/X: follow researchers & writeups
Reporting & Templates
  • Always write clear PoC, steps to reproduce, impact, and remediation
  • Include screenshots, request/response captures, and minimal PoC
  • Respect disclosure rules of each program
Ethics & Legal
  • Only test on systems you own or have explicit permission for.
  • Use official programs and authorized labs; unauthorized testing is illegal.
  • Follow program scope and disclosure timelines.
